インターン9日目 ローカル開発環境構築

Mar 12th, 2013

駒形さん(@komagata)からタスクをいただいたので, 今日は準備としてローカルに怖話の開発環境を構築していた. 途中, 詰まってしまって環境構築だけで一日が終わってしまって残念.

ローカル開発環境を構築する

READMEとコチラを参考に.

$ rbenv install 2.0.0-p0
$ brew install imagemagick mysql qt
$ git clone git@github.com:fjordllc/[怖話]
$ cd [怖話]
$ bundle install --path vendor/bundle
$ cp config/database.example.yml config/database.yml

developmentとtest環境のパスワードをconfig/database.ymlに追記する. まだmySQLの設定をしていなかった場合は, 先に下を参照して行なっておく.

development:
  adapter: mysql2
  encoding: utf8
  pool: 5
  host: localhost
  database: kowabana_development
  username: root
  password: [ここに追加]
  reconnect: true
  socket:

終わったら

$ bundle exec rake kowabana:setup

すればいい…はずがUnknown database 'kowabana_development'というエラーが出て失敗する.

mysql> CREATE DATABASE `kowabana_development` DEFAULT CHARACTER SET `utf8` COLLATE `utf8_unicode_ci`;

として直接データベースを作ってやって,

$ bundle exec rake db:migrate
$ bundle exec rake db:seed

とやってやればいいみたい. 終わったら

$ bundle exec rails s

として起動すればおっけー.

テストの前には同様に

mysql> CREATE DATABASE `kowabana_test` DEFAULT CHARACTER SET `utf8` COLLATE `utf8_unicode_ci`;

としてテスト用のデータベースを作成しなければいけないことに注意. テストは

$ bundle exec rake
$ bundle exec rake konacha:run

で実行できる. テストが通らない場合は以下を試してみるといいらしい.

$ bundle exec rake kowabana:reset
$ bundle exec rake

mySQL

$ brew install mysql

後に

$ brew info mysql

でも表示される指示にしたがって

$ unset TMPDIR
$ mysql_install_db --verbose --user=`whoami` --basedir="$(brew --prefix mysql)" --datadir=/usr/local/var/mysql --tmpdir=/tmp/
$ mysql.server start
$ mysqladmin -u root password 'new-password'

としておく.

OS起動時に自動起動するようにするにはしていないが, このあたり(参照1, 参照2)を見れば良さそう.

怖話の改善案考える

Mar 11th, 2013

今日から作業週. まずはじめに怖話を一通り使ってみて, 気になったこと, こうした方がいいと思ったことなど. 基本的にソースは自分だし, 過去ログも見ていないので見当違いなことも多々あると思う. ごめんなさい.

怖話とは

合同会社フィヨルドが運営するユーザ投稿型の怖い話まとめサイト. 話ごとに「怖い」やコメントをつけることができる. iPhone, Androidアプリもある.

方針

ユーザ投稿型ではあるけれど, ユーザの大多数は読むのがメインで, 時々「怖い」ボタンを押したりコメントをつけたりするだけだと思う(怖い話の投稿はしない). ので, 「どうしたらもっとたくさんの話を読みやすくなるか」という視点で使ってみた.

機能

使っている中で欲しいな, と思ったものを箇条書きで.

  1. ランダムで怖い話を読むボタンは, 全話からランダムではなく人気の話からランダムに表示してほしい. 特に初めてアクセスしたときに, 最初にアレな話が来ると離脱しそう.
  2. 週間ランキング等から全てのランキングを表示するリンクが欲しい.
  3. 週間ランキング等, 個別のランキングページでも10話しか表示されておらず, 全てのランキングページと同じ表示数なのであえて個別に見るメリットが無い. 11位以下を見る方法が欲しい.
  4. ランキングや特選に載っている話を順番に連続で読める機能が欲しい.
  5. ユーザ一覧でソートしたい. 投稿数とか怖いをつけた数とか. で, 上と同様に順番に連続で読みたい.
  6. メールアドレスでメンバ登録したときに, welcomeメールが欲しい.
  7. メンバ登録するメリット(履歴管理が出来るとか)を知りたい(文言か別ページか).
  8. タグはカンマ区切りではなく, 個別に入力したい.
  9. 怖い話を読み飛ばしてしまったときなど, 戻りたい.
  10. [iPhone] web版にアクセスした場合, アプリに誘導するリンクがほしい(cf: Dropboxのモバイル用ページ).
  11. [iPhoneアプリ] ページ遷移の際にLoading…的な反応がほしい. 固まっているように見えるので.

デザイン

素人考えのジャストアイディアですが.

  1. [iPhone]トップページのファーストビューから怖い話への導線がランダムボタンしかないように見えるので, 特選やランキングをスクロールしないでも存在が分かるように移動してほしい.
  2. メニュー項目で「”怖い”話を検索」や「新着の”怖い”話」のように”怖い”が沢山付いているのがシュールに感じる. 無くてもいいのでは.
  3. メニューの表示順は「投稿する > 怖い話を探す」より「怖い話を探す > 投稿する」の方が使用頻度としては良いのでは.
  4. メニューに素材一覧を表示するのは舞台裏を覗いてしまったようで気になる. 投稿ページのみに隠しておいた方がいいのでは.
  5. 怖い話を読み終わった後に表示されるライトボックスで, 「違う話を読む」の項目に「コメントする」というのがあって気になる. ランキングへのリンクの方が自然だと思う.
  6. コメント欄が怖い話から位置的に遠いと感じる. ユーザの動きは「怖い」ボタン->コメントだと思うので, その順番に並べてほしい.

バグ？

  1. 02:00頃にアクセスしたとき, 日間ランキングが表示されなかった. 未集計ということなら前日分を表示してほしい.

自分が出来ること

(やっていいなら)機能の2, 3, 7~10は現在のチュートリアル進行状況でも出来そう. 1, 4~6はRailsの勉強もする必要があると思うので次回以降の作業週になるか. 学習しながらでもよければ1は出来ると思う.

準備

  • VM名: Debian64
  • OS type: Linux
  • OS version: Debian (64 bit)
  • メモリ: 384MB
  • virtual hard driveを作るか: create a virtual hard drive now
  • hard driveのfile type: VDI (VirtualBox Disk Image)

Debianにsshのインストール

Mar 8th, 2013

Debianでsudoを使えるように,Debianのコンソールで日本語を表示の続き.

インストール自体は1行目のみ. 2行目で設定ファイルのバックアップをとって, 3行目で編集する.

$ sudo apt-get install ssh

$ sudo cp /etc/ssh/sshd_config /etc/ssh/sshd_config.org
$ sudo vi /etc/ssh/sshd_config

ここではとりあえずrootでのログインを禁止しておくだけ. その他の設定はさくらのVPSで最初にやっておくことを見てほしい.

PermitRootLogin yes
->
PermitRootLogin no

VMだしPort変更とかはする必要無いと思う. 終わったら

$ sudo /etc/init.d/ssh restart

して設定を有効化しておく.
